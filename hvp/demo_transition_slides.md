# Transition Slide: Choose Your SMAIRT Starting Point

## Three demo paths, one workflow

**1. HVP demo — database-backed science**
- Start with a built PostgreSQL database and rich biological context
- Ask questions about phage-host links, evidence methods, geography, and gene function
- Best for: learning how to turn a real scientific database into iterative analyses

**2. Lunar demo — structured example project**
- Start from a simpler worked example with clearer boundaries
- Practice the SMAIRT loop without first navigating a large domain schema
- Best for: seeing the mechanics of question → script → result → interpretation

**3. Bring your own project — apply the pattern**
- Start with your own data, question, or half-formed hypothesis
- Use SMAIRT to scaffold the project, preserve context, and build a breadcrumb trail
- Best for: converting an existing research idea into reproducible AI-assisted iterations

**Through-line:** each path uses the same discipline — small question, numbered script, logged output, interpretation, next question.

# Getting Started Slide: First 10 Minutes

## Do this before the first analysis

**1. Pick a path**
- HVP if you want a real database demo
- Lunar if you want a simpler guided example
- Bring-your-own if you have a dataset or question ready

**2. Open the project in VS Code**
- Confirm the folder has `background/`, `experiments/`, `results/`, and `prompts/`
- Make sure the AI assistant can see the files it needs

**3. Prime the assistant**
- Ask it to read the background and conventions first
- Require a plan before code
- Keep the first analysis small

**4. Run one reproducible loop**
- Create one numbered script
- Print to console and `results/logs/`
- Paste output and interpretation back into the script

**Goal:** leave with one complete breadcrumb trail, not a perfect final result.
